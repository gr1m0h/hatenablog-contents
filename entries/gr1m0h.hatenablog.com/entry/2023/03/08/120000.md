## はじめに

この記事は、Wezterm の Dracula テーマのドキュメントに対して行ったコントリビュートについてまとめたものです。
1ヶ月ほど前にマージされたのですが、結果としてはやや不本意な形になってしまいました。せっかくなので、経緯を記録として残しておこうと思います。

## 背景

私は普段ターミナルとして Wezterm を使用しています。また、エディタやターミナルなど、テーマを設定できるものにはすべて Dracula テーマを適用しています。そのため、Wezterm にも Dracula テーマをインストールしました。
導入時にドキュメントを確認したところ、記載通りに設定しても動作しない箇所や、不要な記述が含まれている箇所がありました。気になったので、プルリクエストを提出することにしました。

[https://github.com/dracula/wezterm/pull/6:embed:cite]

## 修正内容

修正したのは以下の3点です。

1. Wezterm 公式ドキュメントへのリンクを追加（該当セクションに欠けていたため）
1. color_scheme 設定時のタイポを修正（= ではなく : が正しい）
1. dracula/wezterm とは無関係な wezterm.lua の記述を削除

元の変更コミットは以下です。

[https://github.com/dracula/wezterm/pull/6/commits/0f5c4d6be4835a691f1e7d0703803441c3f4e2bb:embed:cite]

## マージ後の状況

最終的にマージされた内容は以下で確認できます。

[https://github.com/dracula/wezterm/pull/6/files:embed:cite]

結果として、2 と 3 の修正は含まれていませんでした。
念のため補足しておくと、私の変更がすべて採用されなかったこと自体に不満があるわけではありません。2 については、知識のない方が同じミスをする可能性があることを懸念していたのですが、別の方のコミットとして修正されたようなので、結果としては問題ありません。

一方で、3 については残念ながらそのままになっています。以下の設定例において、下3行（`tab_bar_at_bottom` 〜 `window_decorations`）はカラーテーマとは無関係な設定です。

```lua
return {
    color_scheme = "Dracula (Official)",
    tab_bar_at_bottom = true,
    use_fancy_tab_bar = false,
    window_decorations = "RESIZE"
}
```

Dracula テーマのドキュメントとしては、`color_scheme` の設定だけで十分だと考えています。無関係な設定が含まれていると、初めて導入する方が「これも必要なのか」と混乱する可能性があります。とはいえ、動作に支障があるわけではないので、このままでも問題はないと思います。

## さいごに

OSS へのコントリビュートでは、提出した修正がすべて採用されるとは限りません。今回はそのことを改めて実感する経験になりました。
ドキュメントの改善は地味ですが、初めて使う人の体験に直結する部分だと思っています。気になる点を見つけたら、今後も積極的にプルリクエストを出していきたいと考えています。
